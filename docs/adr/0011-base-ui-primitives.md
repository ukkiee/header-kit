# UI 프리미티브는 Base UI로 일원화한다

팝업·탭 앱의 상호작용 프리미티브를 Base UI(@base-ui-components/react) 한 계층으로 통일한다. 셀렉트·칩 그룹·접이식 패널·입력·체크박스·필드는 Base UI로 교체하고, 툴팁과 토스트를 신규 도입한다. 기존 사용처(메뉴·토글 스위치·대형 편집기 Dialog)는 유지한다.

- **맥락**: 프리미티브가 세 계보로 갈라져 있었다 — 네이티브(`<select>`: OS 스타일 팝업이 미니멀 UI와 이질적), 커스텀(칩·접이식 패널: 라벨 래핑이 브라우저의 label→첫 컨트롤 호버 전파를 유발해 엉뚱한 칩이 반응하는 버그), 부분 Base UI(메뉴·스위치·다이얼로그). 접근성 시맨틱(role, aria-invalid, 포커스 관리)도 계보마다 품질이 달랐다.
- **결정**: 전면 일원화. 폼 검증은 Base UI Field/Form의 시맨틱(aria-invalid, 인라인 오류)을 쓰고, 칩은 Toggle/ToggleGroup으로 교체해 캡션-컨트롤 분리로 호버 버그를 구조에서 제거한다. 아이콘 버튼의 툴팁, 삭제 실행 취소의 토스트도 같은 계층에서 온다.
- **트레이드오프**: Base UI는 1.0.0-rc.0이라 API 변동 리스크가 있고, 전면 채택은 그 노출면을 키운다. 대신 스타일·키보드·스크린리더 품질이 한 지점에서 관리되고, 검증·툴팁·토스트를 자체 구현하지 않는다. 대안이었던 표적 교체(문제 지점만)는 세 계보 혼재를 남겨 이 결정의 핵심 이득(단일 관리 지점)을 포기한다.
- **경계**: 드래그 재정렬은 Base UI 영역이 아니다 — dnd-kit이 담당하고, 드래그 애니메이션은 dnd-kit transform에 위임한다(motion과 이중 적용 금지).
