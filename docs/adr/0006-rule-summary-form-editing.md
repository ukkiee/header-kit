# 규칙은 읽기 요약으로 보이고, 편집은 폼에서 일어난다

수정(Modification) 목록의 "항시 편집 가능한 인라인 입력 + 선택 시 옵션 확장" 모델(ADR 0004의 테이블 행)을 버린다. 항상 열린 입력 필드들은 읽기 밀도를 해치고, 필드별 즉시 커맨드 전송은 편집 중간 상태를 저장소에 흘리며 스테일 프롭 경합을 만들었다.

- **표시**: 각 규칙은 읽기 전용 요약 행 — 체크박스(활성) + 이름(메모 우선, 없으면 헤더 이름) + 종류 배지 + 한 줄 효과 요약(`X-Test: aaa`, `pattern → substitution` 등) + `Edit`/`Delete` 텍스트 버튼.
- **생성/편집**: 목록의 `Add rule` 버튼 하나. 폼에서 종류를 고르면 그 종류의 필드가 나타나고, Save가 규칙 전체를 원자적으로 커맨드 전송한다. Cancel은 초안을 버린다.
- 요약 문자열 생성은 순수 함수로 분리해 단위 테스트한다.
- 편집 폼 상태(무엇을 편집 중인가 + 초안)는 ProfileSection 로컬 — 프로필 전환(재마운트)이 초안을 버리는 것이 올바른 의미론이라 앱 레이어 뷰 상태 규칙의 의도적 예외다.
- 아이콘은 lucide 단일 패밀리(strokeWidth 1.75)로 통일하고 유니코드 글리프를 퇴역한다.

트레이드오프: 값 하나 고치는 데 클릭이 한 번 늘어난다(Edit 진입). 대신 목록이 스캔 가능해지고, 저장 전 중간 상태가 저장소에 새지 않으며, 순차 편집 대기 규약이 필요 없어진다.
