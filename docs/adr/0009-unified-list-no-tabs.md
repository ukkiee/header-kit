# 탭을 없애고 규칙과 적용 조건을 한 목록에 둔다

규칙 단위 URL 필터(ADR 0007/0008)가 생기면서 수정/필터 탭 분리는 남은 프로필 조건 9종을 위한 과한 구조가 됐다. 언더라인 탭(ADR 0004의 결정)을 퇴역시킨다.

- **단일 목록**: 규칙(수정) 행들 → `적용 조건` 구분 캡션 → 프로필 조건 행들. 조건도 규칙과 같은 요약 행(체크박스+제목+FILTER 배지+효과 한 줄+Edit/Delete) — 한 화면, 한 패턴.
- **단일 폼**: `Add rule`의 종류 셀렉트가 optgroup(규칙/적용 조건)으로 조건 10종을 포함한다. 조건 필드는 기존 FilterEditor를 폼 안에서 재사용하고, Save가 add/update-filter를 원자 전송한다.
- 조건 요약 생성은 순수 함수(filterView)로 분리해 단위 테스트한다.
- 앱 레이어의 활성 탭 뷰 상태·ui/tabs 프리미티브는 소비자가 없어져 퇴역한다.
- 도메인 의미론 불변: 조건은 여전히 프로필 수준(그 프로필의 모든 규칙을 게이트)이고, 규칙 자체 URL 필터는 그 규칙의 URL 스코프를 대체한다(0007).

트레이드오프: 규칙이 많은 프로필에서 조건이 스크롤 아래로 밀린다 — 구분 캡션과 FILTER 배지가 스캔을 돕는다.
