# 쿠키 수정은 헤더 레벨로만 한다

MV3 declarativeNetRequest는 헤더 값의 부분 수정을 지원하지 않는다(통짜 set/remove/append만 가능). 쿠키 저장소(chrome.cookies)를 고쳐 부분 수정을 재현하는 대안이 있었으나, 적용 범위가 요청 단위가 아니라 브라우저 전역·영속이 되어 "켜진 동안, 스코프 안에서만 변조하고 끄면 흔적이 없다"는 제품 계약을 깬다. 따라서 쿠키 기능은 Cookie/Set-Cookie 헤더에 대한 DNR 표현 가능 연산(append·전체 교체·제거)으로 한정하고, 속성 보존 부분 수정과 regex 쿠키 이름 매칭은 스코프에서 제외한다.

## Consequences

- `cookies` 권한을 요구하지 않아 설치 권한 표면이 최소로 유지된다.
- 쿠키 저장소 편집을 나중에 도입하더라도 같은 UI에 섞지 않고 별도 기능으로 명시 분리한다.
