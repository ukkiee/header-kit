# 팝업과 탭 앱은 단일 셸을 쓴다 (ADR 0004의 팝업 분기 대체)

ADR 0004가 도입한 표면별 이원 셸(팝업=칩 스위처+하단 패널, 탭 앱=레일+사이드바)을 버리고, 두 표면 모두 **레일(프로필/백업/환경설정) + 검색 사이드바 + 본문** 단일 셸로 통일한다. Chrome 팝업 최대 크기(800×600) 안에서 팝업을 ~760×580으로 넓혀 같은 셸을 그대로 렌더한다.

- 칩 스위처(ProfileChips)와 팝업 하단 접이 패널 배치는 퇴역한다. 프로필 탐색·검색·관리 기능 접근이 표면과 무관하게 동일해진다.
- 표면 차이는 두 가지만 남는다: 팝업 상단의 "탭에서 열기" 버튼, 크기(팝업 고정 760×580 + 본문 스크롤, 탭 전폭·전고).
- 선택 재조정 불변식·뷰 상태(선택/탭/확장)는 변화 없다 — 이미 두 표면이 공유하던 것.

트레이드오프: 팝업이 커진다(420→760). 한 손 조작감 대신 표면 간 학습 비용 0과 검색·목록 조망을 얻는다 — 다중 프로필 사용자의 주 사용성이 우선이라는 판단.
