# ui-simplify — Spec

Labels: `ready-for-agent` · invariant: feature · 근거: ADR 0004(단일 프로필 뷰 + 탭 구조)

## Problem Statement

확장을 쓰는 개발자 입장에서 팝업이 "복잡한 도구"로 느껴진다. 프로필이 늘수록 카드가 세로로 무한 적층되고, 카드 안에 또 카드(수정/필터 행)가 중첩돼 보더가 화면을 격자로 채우며, 모든 행에 모드 칩·주석 2줄이 항상 펼쳐져 있어 정작 중요한 "어떤 헤더가 어떤 값으로 걸려 있는가"를 한눈에 읽기 어렵다. 색도 프로필 컬러·액센트·경고·삭제·칩 활성색이 동시에 노출돼 시선이 분산된다.

## Solution

레퍼런스 API 클라이언트류의 디자인 언어 — 보더 대신 여백·명도, 적층 대신 탭, 카드 대신 테이블 — 로 두 표면을 재구성한다.

- **팝업(420px)**: 상단 프로필 칩 스위처(on/off 도트) + 선택된 프로필 하나만 표시. 프로필 내부는 언더라인 탭(수정/필터). 행은 컬럼 헤더 + 얇은 구분선의 테이블형이고 기본 1줄, 선택 시에만 옵션이 확장된다.
- **탭 앱(넓은 표면)**: 아이콘 레일(프로필/백업/환경설정) + 사이드바(프로필 목록·검색) + 본문. 본문 편집 컴포넌트는 팝업과 공유한다.
- **테마**: 시스템 연동 다크+라이트, 액센트 1색.

## User Stories

1. As a 확장 사용자, I want 팝업을 열었을 때 선택된 프로필 하나만 보이기를, so that 한 번에 처리할 정보량이 줄어 현재 작업에 집중할 수 있다
2. As a 확장 사용자, I want 상단 칩 행에서 모든 프로필의 on/off 상태를 도트로 보기를, so that 단일 뷰로 바뀌어도 전체 활성 상태를 잃지 않는다
3. As a 확장 사용자, I want 칩 클릭으로 프로필을 전환하기를, so that 스크롤 없이 프로필 사이를 오갈 수 있다
4. As a 확장 사용자, I want 칩 행 끝의 `+`로 새 프로필을 만들기를, so that 프로필 생성 동선이 목록과 같은 곳에 있다
5. As a 확장 사용자, I want 칩 순서가 프로필 우선순위(겹침 시 상단 우선)를 그대로 표현하기를, so that 기존 우선순위 의미론을 새 UI에서도 읽을 수 있다
6. As a 확장 사용자, I want 프로필 내부가 수정/필터 언더라인 탭으로 나뉘기를, so that 한 화면에 한 관심사만 보인다
7. As a 확장 사용자, I want 탭에 항목 개수가 표시되기를, so that 탭을 열지 않고도 규모를 파악할 수 있다
8. As a 확장 사용자, I want 수정 목록이 이름/값 컬럼 헤더를 가진 테이블형이기를, so that 어떤 헤더가 어떤 값인지 세로로 빠르게 스캔할 수 있다
9. As a 확장 사용자, I want 행이 기본 1줄(체크박스+종류+이름+값)이기를, so that 같은 높이에 더 많은 항목이 보인다
10. As a 확장 사용자, I want 행을 선택했을 때만 모드(덮어쓰기/덧붙이기)·빈 값 처리·주석이 확장되기를, so that 세부 옵션이 필요할 때만 밀도를 지불한다
11. As a 확장 사용자, I want 행마다 종류(요청/응답/쿠키/Set-Cookie/CSP/리다이렉트)가 소형 중립 뱃지로 표시되기를, so that 혼합 목록에서도 행 종류를 즉시 식별한다
12. As a 확장 사용자, I want CSP·리다이렉트 같은 비정형 수정도 같은 테이블 행으로 보이고 선택 시 전용 편집 UI가 확장되기를, so that 목록의 시각 언어가 일관된다
13. As a 확장 사용자, I want 긴 값이 행 안에서 truncate되고 확대 편집기는 유지되기를, so that 레이아웃이 깨지지 않으면서 긴 값 편집도 가능하다
14. As a 확장 사용자, I want OS 다크 모드일 때 확장도 다크로 렌더되기를, so that 브라우저·에디터와 시각적으로 이어진다
15. As a 확장 사용자, I want 라이트 모드도 같은 수준으로 다듬어지기를, so that 어느 테마든 일관된 품질을 얻는다
16. As a 확장 사용자, I want 액센트가 1색으로 절제되기를, so that 활성 상태·선택 같은 진짜 신호만 색으로 도드라진다
17. As a 확장 사용자, I want 프로필 이동/복제/삭제가 `⋯` 메뉴로 모이기를, so that 헤더에 아이콘 버튼이 줄지어 있지 않다
18. As a 확장 사용자, I want 프로필 on/off 토글·이름·뱃지 편집이 단일 뷰 헤더에 남기를, so that 자주 쓰는 조작은 메뉴 없이 즉시 가능하다
19. As a 확장 사용자, I want 상태 요약(규칙 수·활성 프로필·경고)이 상단 슬림 라인으로 유지되기를, so that 지금 브라우저에 무엇이 걸려 있는지 항상 보인다
20. As a 확장 사용자, I want compile 경고가 여전히 지역화된 라벨+상세로 보이기를, so that 문제 원인을 언어 장벽 없이 파악한다
21. As a 확장 사용자, I want 일시정지/재개가 상단에서 한 클릭으로 유지되기를, so that 긴급히 전체를 끌 수 있다
22. As a 확장 사용자, I want 내보내기/가져오기/백업/환경설정이 팝업 하단에 남되 시각적으로 정돈되기를, so that 기능 접근성을 잃지 않으면서 본문이 가벼워진다
23. As a 넓은 화면 사용자, I want 탭 앱에서 사이드바 프로필 목록과 검색을 쓰기를, so that 프로필이 많아도 빠르게 찾는다
24. As a 넓은 화면 사용자, I want 아이콘 레일에서 백업·환경설정 화면으로 전환하기를, so that 관리 기능이 본문 편집과 분리된다
25. As a 넓은 화면 사용자, I want 탭 앱과 팝업의 편집 경험이 동일하기를, so that 표면을 오갈 때 새로 배울 것이 없다
26. As a 키보드 사용자, I want 탭·칩·행 확장이 키보드와 스크린리더로 조작 가능하기를, so that 접근성이 구조 개편으로 후퇴하지 않는다
27. As a 확장 사용자, I want 420px에서 어떤 상태로도 가로 오버플로가 없기를, so that 이전 피처에서 고친 레이아웃 견고성이 유지된다
28. As a 한국어 사용자, I want 새 UI의 모든 라벨이 en/ko 카탈로그를 거치기를, so that 언어 완전성이 코드로 강제된 채 유지된다

## Implementation Decisions

- **저장 스키마·core·runtime·platform 무변경.** 이 피처는 순수 뷰 재구성이다. compile/summary/i18n 계약(WarningView 등)도 그대로 쓴다.
- **뷰 상태는 비영속 로컬 상태**: 선택 프로필 id, 활성 탭(수정/필터), 확장 행 id는 앱 레이어의 로컬 상태다. 팝업을 열면 첫 활성 프로필(없으면 첫 프로필)이 선택된다. 저장 상태에 넣지 않는다.
- **신규 ui 프리미티브(cva 레시피)**: 언더라인 Tabs, 프로필 칩 스위처, 테이블형 행(헤더 행+데이터 행+확장 영역), 종류 뱃지, `⋯` 드롭다운 메뉴. 기존 Base UI 컴포넌트를 기반으로 하고, 기존 프리미티브(Button/Input/Select/Chip 등)는 재사용한다. 행 카드(Card row variant)는 퇴역한다.
- **테마는 시스템 연동만**: Tailwind `dark:` 변형 + `prefers-color-scheme`. 수동 테마 스위치는 만들지 않는다. 디자인 토큰에 다크 값을 추가하고, 액센트는 1색으로 통일한다.
- **탭 앱 셸은 표면 전용 레이아웃**: 아이콘 레일+사이드바는 넓은 표면에만 존재하고, 본문(프로필 편집)은 팝업과 같은 컴포넌트를 공유한다. 팝업 하단 접이 패널(백업/환경설정/전송)은 탭 앱에서 레일 화면으로 승격된다.
- **행 확장은 단일 확장**: 한 번에 한 행만 확장된다(레퍼런스식 포커스 모델). 다른 행을 선택하면 이전 행은 접힌다.
- **i18n**: 신규 라벨(탭 이름, 메뉴 항목, 컬럼 헤더 등)은 전부 en/ko 카탈로그에 추가 — 키 parity는 기존 방식대로 tsc가 강제한다.
- **조작 가능 요소는 role/aria-label 기반 식별을 유지**: smoke 테스트와 접근성이 같은 셀렉터를 공유한다.

## Testing Decisions

- 좋은 테스트는 외부 행동만 검증한다: "UI를 이렇게 조작하면 실제 요청/상태가 이렇게 변한다". 내부 구현(컴포넌트 트리, 상태 훅)에 결합하지 않는다.
- **Playwright smoke가 최상위 seam** (기존 48항목): 시나리오는 유지하고 조작 경로만 새 구조로 갱신한다(칩 클릭→전환, 탭 클릭→필터, 행 클릭→옵션 확장). 신규 행동 — 프로필 전환, 탭 전환, 행 확장/단일 확장 — 항목을 추가한다. 선행 예: 기존 smoke의 H/M 시리즈.
- **Vitest**: core 161개 무변경 통과가 회귀 가드. 신규 순수 뷰 로직이 생기면 단위 테스트 추가 — 선행 예: 경고 매핑 순수 함수 테스트.
- **Storybook**: 신규 프리미티브의 상태별(다크/라이트, 선택/확장, 개수 뱃지) 스토리 — 선행 예: 기존 ui 프리미티브 스토리.
- **렌더 감사**: 진단 스크립트로 420px+ko 실데이터 스크린샷(다크/라이트 양쪽)을 떠 검증 증거로 남긴다.
- 신규 seam 0개.

## Out of Scope

- 기능 추가·변경 일절 없음: 새 수정/필터 종류, 스키마 변경, DNR 컴파일 로직, 전송 형식 등.
- 하드코딩 영어 aria-label 정리(이전 피처의 알려진 잔여) — 별도 후속.
- 수동 테마 스위치, 커스텀 액센트 색 설정.
- 드래그앤드롭 프로필/행 정렬 — 이동은 메뉴로 충분.
- 애니메이션/모션 디자인 세부(기본 transition 수준만).
- 프로필 검색(팝업) — 사이드바 검색은 탭 앱만.

## Further Notes

- 구조 결정 근거는 ADR 0004. 디자인 토큰 방향은 `docs/design-system.md`와 정합 유지(액센트 1색 원칙으로 갱신 여지).
- 레퍼런스 원본 이미지는 브랜드 중립 원칙에 따라 저장소 밖에 둔다 — 문서·커밋에 레퍼런스 제품명 금지.
- 트레이드오프(전 프로필 동시 조망 상실)는 ADR 0004에 기록된 대로 칩 도트 + 탭 앱으로 보완.
