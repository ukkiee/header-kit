# 02 — 스크롤 영역을 앱 스타일로 통일

**What to build:** 사이드바와 본문의 스크롤바가 OS 기본이 아니라 앱의 시각 언어를 따른다(다크 모드 포함). 프로필이 아무리 많아도 팝업이 가로로 넘치지 않고, 규칙이 많은 프로필에서도 스크롤이 부드럽게 동작한다.

**Blocked by:** 01 — 기준선이 먼저 떠 있어야 이 티켓이 시작 지연에 얼마를 더했는지 말할 수 있다.

**Status:** ready-for-agent

- [ ] 사이드바·본문의 네이티브 세로 스크롤이 앱 스타일 스크롤바로 바뀌고 다크 모드를 따른다
- [ ] 팝업이 고정 크기이므로 스크롤바가 콘텐츠 폭을 잠식하지 않는다
- [ ] 프로필이 많고 이름이 최장일 때 가로 오버플로 0 — 진단의 기존 경계 검사가 계속 통과한다
- [ ] 진단의 "내부 가로 스크롤러 0" 검사가 의도된 스크롤 영역 viewport는 제외하도록 조정되고, **조정 사유가 스크립트에 남는다**. 의도치 않은 가로 스크롤러는 여전히 실패로 잡힌다
- [ ] 번들 델타와 시작 지표 중앙값을 이 티켓 파일에 기록한다 — 여유가 얇아 어느 도입이 무엇을 압박했는지 갈라낼 수 있어야 한다
- [ ] 번들 게이트 PASS. 초과 시 **한도를 올리지 않고** 지연 로드나 대안을 먼저 검토한다
- [ ] 전 게이트 green
