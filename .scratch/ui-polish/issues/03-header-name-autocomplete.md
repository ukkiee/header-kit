# 03 — 헤더 이름 자동완성을 앱 팝업으로

**What to build:** 헤더 이름 자동완성이 브라우저 기본 UI가 아니라 앱의 다른 팝업과 같은 모양·같은 키보드 조작을 갖는다. 내가 등록한 이름이 표준 헤더보다 먼저 뜨고, 입력한 접두에 맞는 후보만 보이며, Esc는 제안 팝업만 닫고 편집 중인 폼은 유지된다.

**Blocked by:** 02 — 번들 여유가 ~3KB로 얇다. 스크롤 영역이 얼마를 먹었는지 알고 시작해야 대안이 필요한지 판단할 수 있다.

**Status:** ready-for-agent

- [ ] 헤더 이름 입력의 제안이 앱 팝업 표면으로 렌더되고 기존 팝업과 같은 표면·항목 토큰을 쓴다
- [ ] 후보 산출 규칙(사용자 항목 우선·대소문자 무시 중복 제거·상한 8개)이 도메인 계층에 그대로 남고 **기존 단위 테스트가 수정 없이 통과한다** — 컴포넌트 자체 필터링에 의존하지 않는다
- [ ] 화살표로 제안을 이동하고 Enter로 고를 수 있다
- [ ] Esc가 제안 팝업만 닫고 폼은 열린 채 유지된다 — 실수로 편집이 취소되지 않는다
- [ ] 번들 델타와 시작 지표 중앙값을 이 티켓 파일에 기록한다
- [ ] 번들 게이트 PASS. 한도를 넘으면 **한도를 올리지 않고** 지연 로드 또는 기존 브라우저 자동완성 유지를 검토하고, 둘 다 불가하면 진행을 멈추고 사용자 재트리아지를 받는다
- [ ] 전 게이트 green
