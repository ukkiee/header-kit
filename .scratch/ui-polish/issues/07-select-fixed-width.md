# 07 — URL 필터 셀렉트 폭 고정

**What to build:** URL 매치 방식을 바꿔도 셀렉트 폭이 그대로라 옆의 패턴 입력이 밀리지 않는다 — 값을 고르는 동안 레이아웃이 흔들리지 않는다. 동시에 가장 긴 라벨도 잘리지 않아 선택한 값을 항상 읽을 수 있고, 한국어 로케일에서도 같다.

폭 안정성만 보는 테스트는 **폭을 좁게 잡아 라벨이 잘려도 통과한다** — 두 단언이 함께 걸려야 한다(스펙 R-2).

**Blocked by:** 01

**Status:** ready-for-agent

- [ ] 매치 방식을 어떤 값으로 바꿔도 셀렉트 트리거 폭이 동일하고 옆 입력이 밀리지 않는다
- [ ] 기본 폭이 공유 스타일 조각의 이름 있는 토큰이고, 호출자가 필요 시 조정할 수 있다
- [ ] smoke: en·ko 각각에서 **모든** 매치 방식 옵션을 순회하며 트리거 폭이 동일(±0.5px)
- [ ] smoke: 각 선택 상태에서 라벨이 잘리지 않는다(`scrollWidth ≤ clientWidth + 1`) — 최장 라벨(en `Regex (advanced)` / ko `정규식 (고급)`)에서도 성립한다
- [ ] 셀렉트 팝업이 기존처럼 앵커 폭 이상으로 열린다
- [ ] 전 게이트 green
