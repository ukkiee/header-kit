## 측정이 설계를 바꿨다 — 고정 폭을 프리미티브 기본값으로 두지 않았다

스펙은 "Select 프리미티브에 폭 제약을 도입하되 최장 라벨이 잘리지 않는 폭을 **기본값**으로"라고 적었다. 그런데 네 개의 Select를 실측해 보니:

| 셀렉트 | 자연 폭 | 내용에 따라 변하나 |
|---|---|---|
| 종류 | 406px | 아니오 — Field가 폭을 정한다 |
| **URL 매치 방식** | **en 132 / ko 94px** | **예 — `shrink-0`으로 같은 행에 있다** |
| 모드 | 199px | 아니오 — 그리드가 정한다 |
| 빈 값 처리 | 199px | 아니오 — 그리드가 정한다 |

**출렁이는 것은 하나뿐이다.** 고정 폭을 기본값으로 두면 나머지 셋이 406·199 → 136px로 좁아진다 — 아무도 요청하지 않은 레이아웃 변경이다. 그래서 `width` 변형(`auto` 기본 / `fixed`)을 두고, 옆을 미는 그 자리에만 `fixed`를 준다. 토큰이 "기본 폭"이라는 스펙의 취지는 그대로다 — 값이 한 곳에 있고 호출자가 고른다.

## 폭 값 8.5rem의 근거

en `Regex (advanced)`가 최장이다: 라벨 102px + 아이콘 12px + 간격 4px + 좌우 패딩 12px ≈ 130px(실측 자연 폭 132px). 여유 4px을 얹어 136px. ko 최장(`정규식 (고급)`)은 64px이라 여유가 넉넉하다.

## 두 방향으로 못박았다

플랜 게이트 R-2가 경고한 것이 "폭 안정성만 보면 폭을 좁게 잡아 라벨이 잘려도 통과한다"였다. 두 퇴화 경로를 각각 되돌려 확인했다.

- `width="fixed"`를 빼면 → 폭 `104.92/72.84/116.34/132.13`, 패턴 좌변 `423.92/391.84/435.34/451.13`으로 **사용자가 보고한 증상 그대로** FAIL
- 토큰을 5rem(80px)으로 좁히면 → 폭은 안정(`80`)이지만 **미절단=false**로 FAIL

## 부수로 고친 것

셀렉트 팝업을 여닫는 동안 Base UI의 inert 백드롭이 다음 클릭을 가로채 스위트가 죽었다. 열림·닫힘을 매번 명시적으로 기다리도록 했다 — `waitForTimeout`으로 눈대중하지 않는다.

## 리뷰 반영 (2축)

**양 축이 짚은 최상위 결함 — 절단 단언이 공허하게 통과할 수 있었다.** 라벨 노드를 위치로(`querySelector('span')`) 고르고 없으면 `-1`을 쓰다 보니 `-1 <= -1 + 1`이 참이었다. 노드가 아이콘 래퍼로 밀리거나 사라지면 **R-2가 지키려던 단 하나의 단언이 조용히 무력해진다.** `.truncate`(라벨에만 붙는다)로 정확히 겨냥하고 노드가 없으면 실패하도록 바꿨다. 선택자를 없는 것으로 돌려 `미절단=false`가 나오는 것까지 확인했다.

**Spec: 스펙이 명시한 Storybook 렌더 게이트가 비어 있었다.** "storybook build — 교체·신규 프리미티브(**Select 폭 변형**, …)의 렌더 게이트"인데 `select.stories.tsx`에 폭 변형 스토리가 없어 빌드만 통과하고 있었다. `FixedWidth`(fixed/auto 나란히)와 `FixedWidthKo`를 추가했다.

**Standards: `w-[8.5rem]` 대신 스케일 값.** Tailwind v4가 `w-34`를 생성한다(`calc(var(--spacing) * 34)` = 136px, 실측 확인). 이 파일은 스케일에 없는 것만 임의값을 쓰는 관례다.

**Standards: 폭을 정하는 채널이 둘이라 조용히 충돌할 수 있었다.** 이 저장소는 className을 병합하지 않고 덧붙이므로(tailwind-merge 없음), 호출자가 `w-40`을 className으로 넘기면 두 폭 클래스가 남아 CSS 소스 순서가 승자를 정한다. 변형 주석에 "폭은 이 변형으로만" 을 명시했다.

**Standards: 토큰 주석의 산술이 2px 어긋나 있었다** — 130px 계산과 실측 132px 사이의 좌우 보더 2px을 빠뜨렸다. 채워 넣었다.

**Standards: `shrink-0`과 `width="fixed"`가 3줄 간격으로 같은 의도를 두 번 말하고 있었다.** 리뷰는 `shrink-0`이 이제 지지하지 않는다고 봤지만(형제가 `flex-1 min-w-0`이라 압박이 없다), 좁은 자리에서의 방어로 남기고 역할을 같은 주석에 흡수했다.

**Standards: 스모크 헬퍼가 로케이터를 밖으로 흘렸다.** 호출부가 여닫는 동작을 한 번 더 흉내 내며 대기 규율이 두 곳으로 갈라져 있었다. 선택 직후 호출되는 probe 콜백을 받게 해 한 번의 순회로 끝낸다.

**반영하지 않은 것** — 다른 세 셀렉트의 미절단은 단언하지 않는다. 스펙의 Testing Decisions가 매치 방식으로 범위를 잡았고, 실측상 여유가 넉넉하다(라벨 ~70px / 셀 199px). 다만 이들도 `truncate`를 쓰므로 라벨이 크게 길어지면 조용히 말줄임될 수 있다 — 릴리스 게이트가 볼 수 있도록 여기 남긴다.
