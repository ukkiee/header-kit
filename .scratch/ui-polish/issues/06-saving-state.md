# 06 — 저장 중 상태

**What to build:** 저장을 누르면 눌린 것이 반영 중임이 보인다. 반영 중에는 같은 저장이 두 번 나가지 않고, 성공하면 폼이 자연스럽게 접히며, 거부되면 폼이 남아 고친 뒤 다시 시도할 수 있다. 저장은 배경 작업자와의 왕복이라 지연이 실재한다 — 지금은 그 창에 아무 표시가 없다(스펙 R-3).

**Blocked by:** 01

**Status:** ready-for-agent

- [ ] 저장이 진행 중인 동안 저장 버튼 라벨이 카탈로그의 새 키로 바뀐다 (en `Saving…` / ko `저장 중…`)
- [ ] 진행 중에는 저장·취소 버튼이 모두 비활성이고, 재클릭이 명령을 추가로 보내지 않는다
- [ ] 성공하면 폼이 닫히고, 닫힘이 기존 높이 전이를 타 자연스럽게 접힌다
- [ ] 거부되면 라벨이 저장으로 복귀하고 버튼이 다시 눌리며, 폼은 열린 채 초안이 보존되고 거부 메시지가 alert로 노출된다
- [ ] smoke: 왕복을 인위적으로 지연시켜 위 상태들을 관측한다. **프로덕션 코드에 테스트 전용 훅을 심지 않는다**
- [ ] smoke: 지연 중 재클릭해도 저장 명령이 1회, 스토리지에 규칙이 1건만 반영된다
- [ ] smoke: reduced-motion에서 취소·저장 직후 폼 노드가 exit 창 없이 DOM에서 제거된다
- [ ] 새 카탈로그 키가 en/ko 양쪽에 있고 i18n 커버리지 테스트가 통과한다
- [ ] 전 게이트 green
