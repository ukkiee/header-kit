# 08 — 검증 실패 시 첫 누락 입력으로 포커스

**What to build:** 필수 항목을 비운 채 저장하면 오류만 보이고 끝나는 게 아니라, 고쳐야 할 그 입력으로 포커스가 이동해 손을 옮기지 않고 바로 타이핑할 수 있다. 누락이 여러 개면 위에서부터 차례로 고칠 수 있도록 첫 번째 항목으로 간다.

**Blocked by:** 01

**Status:** ready-for-agent

- [ ] 저장이 검증으로 막히면 첫 누락 필수 입력으로 포커스가 이동하고, 그 자리에서 바로 타이핑된다
- [ ] 종류별 매핑이 렌더 분기와 1:1 — 헤더/쿠키 이름, Redirect 패턴, Redirect 치환, CSP는 첫 디렉티브 이름
- [ ] Redirect의 패턴·치환이 둘 다 비면 자연스러운 입력 순서를 따라 패턴으로 간다
- [ ] smoke: 종류별로 현재 포커스가 기대한 입력임을 확인한다 (헤더 이름·Redirect 패턴·CSP 디렉티브 이름)
- [ ] 기존 인라인 오류 표시가 그대로 유지된다 — 포커스 이동이 오류 표시를 대체하지 않는다
- [ ] 전 게이트 green
